Cybersecurity Essentials

Password Hygiene

  • Use unique passwords per account
  • Prefer 14+ characters, high entropy
  • Avoid patterns (1234, qwerty, dates)
  • Rotate compromised credentials quickly

Threat Modeling

  • Assume device loss ‚Äì use PIN & encryption
  • Reduce attack surface (no server sync)
  • Mitigate shoulder surfing (mask toggle)
  • Clipboard hygiene (copy only when needed)

Browser Security

  • Keep browser up to date (patch cycle)
  • Limit invasive extensions
  • Use strict site permissions
  • Clear sensitive sessions regularly

Responsible Use

  • Never reuse master PIN
  • Prefer hardware-encrypted disks
  • Lock your OS when AFK
  • Enable 2FA everywhere possible

Architecture Snapshot

Storage Layer

Passwords stored in IndexedDB; each entry encrypted with per-PIN derived key (PBKDF2 / subtle crypto). No plain text persists.

Crypto Layer

Randomness sourced from crypto.getRandomValues; AES-GCM ensures confidentiality & integrity (nonce + auth tag).

Privacy Model

Offline-first: no network calls for generation or decryption; export actions require explicit user consent.

Extensibility

Future modules can add breach-check APIs (opt-in) or hardware token integration while preserving core isolation.
